# 설치가이드 — gesture_kiosk 추론 (윈도우 + NVIDIA GPU)

다른 윈도우 PC에 그대로 옮겨 실행하기 위한 가이드입니다.
명령은 전부 **이 폴더(gesture_kiosk) 안에서** 실행합니다.

| 항목 | 기준 |
|---|---|
| OS | Windows 10/11 64비트 |
| GPU | NVIDIA (RTX 30·40·50시리즈 — CUDA 12.8 휠 사용) |
| GPU 드라이버 | 최신 게임레디/스튜디오 드라이버 (CUDA 12.8 지원 버전) |
| Python | **3.11.5** (설치 시 "Add python.exe to PATH" 체크) |
| 카메라 | USB 웹캠 1대 (오토포커스 권장) |
| 마이크 | 불필요 (입력은 제스처·주민등록증뿐) |

> ⚠ TensorRT 가속을 켜면(`use_tensorrt: true`) 첫 실행 때 **이 PC 전용 엔진 캐시**가
> 생깁니다(models/weights/trt_cache/). 다른 PC 것을 복사해 쓰지 마세요 — GPU
> 기종·드라이버에 묶여 있어 동작하지 않습니다.

---

## A. 인터넷이 되는 PC — 3단계

```bat
:: 1) 이 폴더를 통째로 복사해 온 뒤, 폴더 안에서:
install.bat

:: 2) 실행 (브라우저가 http://localhost:5000 을 열면 데모 화면)
run.bat

:: 3) (선택) 최고 성능 — configs\config.yaml 에서 use_tensorrt: true
::    (첫 실행 때 몇 분간 이 PC 전용 엔진 캐시를 만든 뒤부터 빨라짐)
```

`install.bat`이 하는 일: Python 3.11 확인 → 가상환경(venv_win) 생성 →
torch(EasyOCR용)·onnxruntime-gpu·고정 버전 패키지 설치 → 포즈 모델 캐시 → 설치 검증(smoke_test).

> 📄 **라이선스 (C안, 2026-07-10)**: 기본 제스처 엔진은 MediaPipe(Apache-2.0)로,
> 저작자 표시 등 고지 의무가 없습니다. 구 HaGRID(YOLOv10) 엔진은 AGPL 리스크로
> **납품 사용 금지** — 사용 시에만 `models/weights/NOTICE_HaGRID.md` 고지 필요 (README 참고).

## B. 내부망(인터넷 차단) PC — 번들 반입

정부 민원발급기처럼 인터넷이 없는 PC는, **인터넷 되는 윈도우 PC에서 번들을 만들어**
통째로 반입합니다.

```bat
:: [인터넷 되는 윈도우 PC — 같은 Python 3.11]  이 폴더 안에서:
make_offline_bundle.bat
::  → wheelhouse\ (파이썬 패키지 휠 전부) + bundle_models\ (가중치 + EasyOCR 한국어 모델) 생성

:: 프로젝트 폴더 전체를 zip → USB/반입절차로 대상 PC에 복사

:: [대상 내부망 PC]  폴더 안에서 (wheelhouse가 있으면 자동으로 오프라인 설치):
install.bat
run.bat
```

> 번들 제작 PC와 대상 PC는 **같은 Python 3.11 / 64비트 윈도우**여야 합니다.
> (pip이 제작 PC 기준으로 휠을 받기 때문)

## C. 실행 모드

| 명령 | 용도 |
|---|---|
| `run.bat` | 파이프라인 + 예시 UI (시연용, http://localhost:5000) |
| `run.bat --headless` | 파이프라인만 — 이벤트는 config `event_output` 대로 전송 (회사 프로그램 연동 시) |
| `venv_win\Scripts\python scripts\benchmark.py` | 추론 단독 FPS 측정 (KPI 30 FPS 확인) |

## D. 회사 키오스크 프로그램(UI) 연동 계약

엔진과 UI는 아래 3개 통로로만 만납니다. 자세한 예시는 `demo_ui/index.html` 참고.

| 방향 | 통로 | 내용 |
|---|---|---|
| 엔진→UI | `event_output` (udp: JSON) 또는 `/data` 폴링 | `move_left` `move_right` `select` `go_home` `fill_id_fields`(이름·주민번호) + 레거시 이벤트 |
| UI→엔진 | `POST /announce {"text": "..."}` | 포커스된 항목 등 화면 안내를 음성으로 읽어달라는 요청 |
| UI→엔진 | `POST /ocr/start` / `POST /ocr/stop` | 본인확인 화면 진입 시 주민등록증 인식 시작/중단 |

- 이동은 좌/우뿐입니다. **줄 끝에서 다음 줄 첫 칸으로 넘기는 랩(토크백식 선형 순회)은 UI 책임**입니다.
- 새 수신 규격(소켓/시리얼 등)이 정해지면 `src/pipeline/event_sender.py`에 Sender 1개를 추가하고
  config `event_output.mode`로 선택합니다 — 파이프라인 수정 불필요.

## E. 동작 기준 (2026-07-15 개편 스펙 — 장애인·비장애인 범용 설계)

| 동작 | 판정 | 이벤트 |
|---|---|---|
| 포커스 이동 | 팔을 좌/우로 쓸기 (포즈 손목 궤적 — 손이 없어도 동작) | `move_left` / `move_right` |
| 이전 화면 | 팔을 아래로 쓸기 | `go_back` |
| 처음으로 | 팔을 위로 쓸기 | `go_home` |
| 선택·확인 | 손등(또는 팔등)을 카메라로 보이며 유지 | `select` |
| 사용자 고정 | 초점 맞은(선명·큰) 얼굴 기준 잠금 — 다른 사람 손 무시 | (이벤트 없음 — 상태) |
| 주민등록증 | 카메라 제시 → 이름·주민번호 자동 입력 | `fill_id_fields` |

팔등 판정은 자체 학습 모델(models/weights/arm_side_cnn.onnx)이 있을 때만 동작합니다 —
없으면 손등 판정만으로 동작하며, scripts/collect_arm_side.py(수집) →
scripts/train_arm_side.py(학습)로 만듭니다. 구 동작(주먹→펴기·OK핀치·양손바닥
10초)과 레거시 토글은 2026-07-15 제거됐습니다.

## F. 개인정보(주민등록번호) 처리 원칙

- 카메라 프레임·인식 결과를 **디스크에 저장하지 않습니다** (전 과정 메모리 처리)
- 로그에는 **마스킹 값만** 남습니다: `홍*동` / `950101-1******`
- 인식 원문은 `fill_id_fields` 이벤트로 UI에 전달된 뒤 엔진에 남지 않습니다
- ⚠ **주민등록번호 처리의 법적 근거(개인정보보호법 제24조의2)는 회사 확인 필요** — docs/TODO.md №11

## G. 문제 해결

| 증상 | 조치 |
|---|---|
| `install.bat`에서 Python을 못 찾음 | Python 3.11.5 설치 시 PATH 체크 누락 — 재설치 또는 `py -3.11` 확인 |
| 스모크 테스트 "GPU 가속" FAIL | NVIDIA 드라이버 업데이트, `pip list`로 onnxruntime-gpu 설치 확인. 노트북이면 NVIDIA 제어판에서 고성능 GPU 지정 |
| 카메라가 안 열림/수십 초 걸림 | config `camera.device_id` 0→1 변경, `camera.windows_backend`를 msmf↔dshow로 바꿔 시도 |
| 캡처 FPS가 ~5에 묶임 | `windows_backend: msmf` 확인 (dshow는 카메라에 따라 무압축 YUY2로만 협상돼 USB 한계에 걸림) |
| 한글 음성이 안 나옴 | 윈도우 설정 → 시간 및 언어 → 음성 → 한국어 음성(Heami 등) 설치 |
| OCR이 인식을 못 함 | 조명 확보, 카메라에 카드를 화면 안내 영역(중앙)에 크게. `ocr.min_text_conf` 낮추기 |
| 30 FPS 미달 | config `use_tensorrt: true`(첫 실행 시 캐시 생성) 또는 `input_size_px` 640→480 |
| 다른 사람 손에 반응 | `person_lock.wrist_match_ratio`를 줄인다 (기본 0.14) |

## H. 폴더 구조 (추론)

```
gesture_kiosk/
├─ install.bat / run.bat / make_offline_bundle.bat   # 이식·실행 진입점
├─ 설치가이드.md                                      # 이 문서
├─ requirements.txt          # 고정 버전 (재현성)
├─ configs/config.yaml       # 모든 튜닝값 — 수정은 여기서만
├─ models/weights/           # 제스처 ONNX + NOTICE_HaGRID.md(고지문) + trt_cache(이 PC 전용)
├─ src/
│   ├─ capture/              # 카메라 캡처 스레드
│   ├─ inference/            # 제스처 검출 + 사람 포즈
│   ├─ postprocess/          # 사용자 잠금 + 동작 판정(FSM)
│   ├─ ocr/                  # 주민등록증 판독 (EasyOCR)
│   ├─ announce/             # 음성 안내 (토크백 TTS)
│   ├─ pipeline/             # 실시간 루프 · 이벤트 전송 · 예시 UI 서버
│   └─ utils/
├─ scripts/                  # run_demo · download_weights · benchmark · smoke_test · export_onnx(개발용)
├─ demo_ui/                  # 예시 민원발급기 화면 (회사 UI 수령 시 교체)
└─ tests/                    # 단위 테스트 (카메라·모델 없이 실행 가능)
```
