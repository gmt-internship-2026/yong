# 설명서 — gesture_kiosk 실행·사용법

이 문서 하나로 "어떻게 켜고 어떻게 쓰는지"를 확인할 수 있게 정리했습니다.
설치 절차의 세세한 내용(내부망 반입, 문제 해결 전체 표)은 **[설치가이드.md](설치가이드.md)**를 참고하세요.

## 1. 이게 뭔가

USB 웹캠 1대로 얼굴을 인식해 손 없이 화면을 조작하는 배리어프리 민원발급기 데모입니다.
GPU 없이 CPU만으로 동작합니다.

| 동작 | 방법 |
|---|---|
| 포인터 이동 | 코끝을 화면 커서로 — 고개를 움직이면 커서가 따라간다 |
| 선택 | 입 벌리기(즉시) 또는 커서를 1.5초 가만히 두기(응시) — 링 모양 진행 표시가 뜬다 |
| 뒤로(단계별) | 양 눈을 감고 0.6초 버티기 — 화면 상단 진행 바가 채워진다. 직전 단계로 돌아가며
                입력값(주민등록번호·장수)은 그대로 유지된다 |
| 처음으로 | 발급 완료 화면에서는 뒤로가기가 곧 처음으로 (더 되돌아갈 단계가 없어서) |
| 커서 재정렬 | 코를 찡그리기(찡긋) — 자리를 옮기지 않고도 커서 중심을 지금 고개 위치로 다시 잡는다 |

> 2026-07-20: 입 벌림·눈 감김 판정은 고정 수치가 아니라 **잠금 직후 사용자의
> 평상시 표정을 기준선으로 캡처**해 그로부터 얼마나 벗어났는지로 판정합니다 —
> 사람마다 평상시 표정 편차가 있어도(계기판에 "기준" 수치로 표시) 오탐·미탐이
> 줄어듭니다.

## 2. 처음 한 번만 — 설치

이 폴더(`gesture_kiosk`) 안에서:

```bat
install.bat
```

Python 3.11 확인 → 가상환경(`venv_win`) 생성 → 패키지 설치 → 얼굴 랜드마크 모델
다운로드(`models/weights/face_landmarker.task`) → 설치 검증(스모크 테스트)까지 자동으로
됩니다. 인터넷이 안 되는 PC는 [설치가이드.md](설치가이드.md)의 "B. 내부망(인터넷 차단) PC" 절을 보세요.

## 3. 실행하는 법 (전부)

### 3-1. 데모 실행 (가장 많이 쓰는 것)

```bat
run.bat
```

브라우저에서 `http://localhost:5000` 접속 → 실시간 캠 화면 위에 키오스크 메뉴가
얹혀 나옵니다. 종료는 이 창에서 `Ctrl+C`.

### 3-2. 헤드리스 실행 (UI 없이 이벤트만)

회사 프로그램과 실제로 연동할 때처럼, 화면 없이 판정 결과만 `event_output` 설정대로
내보냅니다(기본은 콘솔 로그).

```bat
run.bat --headless
```

### 3-3. 직접 파이썬으로 실행 (run.bat 안 거치고)

`venv_win`을 이미 만든 상태라면, 가상환경을 활성화한 뒤:

```bat
venv_win\Scripts\activate.bat
python scripts\run_demo.py
python scripts\run_demo.py --headless
python scripts\run_demo.py --config configs\config_mac.yaml   :: 맥 등 다른 설정 파일 사용 시
```

활성화 없이 한 줄로 실행하려면 가상환경의 python을 직접 지정합니다:

```bat
venv_win\Scripts\python.exe scripts\run_demo.py
```

### 3-4. 단위 테스트 (카메라·모델 없이 판정 로직만 검증)

```bat
venv_win\Scripts\python.exe -m unittest discover tests -v
```

### 3-5. 설치 검증 (스모크 테스트)

카메라 없이, 패키지가 제대로 깔렸는지 + 더미 프레임으로 얼굴 랜드마크 모델이
로드되는지 확인합니다. `install.bat` 마지막 단계에서 자동으로도 실행됩니다.

```bat
venv_win\Scripts\python.exe scripts\smoke_test.py
```

### 3-6. 추론 FPS 벤치마크 (실기 성능 측정, 카메라 필요)

```bat
venv_win\Scripts\python.exe scripts\benchmark.py                    :: 더미 프레임 1000장
venv_win\Scripts\python.exe scripts\benchmark.py --source camera    :: 실제 카메라로 측정
venv_win\Scripts\python.exe scripts\benchmark.py --frame-count 300  :: 프레임 수 조절
```

### 3-7. 얼굴 랜드마크 모델만 다시 받기

`install.bat`이 자동으로 하지만, 모델 파일만 따로 받고 싶을 때:

```bat
venv_win\Scripts\python.exe scripts\download_weights.py
```

### 3-8. 내부망(오프라인) 설치 번들 만들기

**인터넷 되는** 같은 사양(윈도우 + Python 3.11) PC에서:

```bat
make_offline_bundle.bat
```

`wheelhouse\`(패키지 휠) + `models\weights\face_landmarker.task`가 만들어집니다.
이 폴더 전체를 zip으로 묶어 대상 PC로 옮긴 뒤, 대상 PC에서 `install.bat`만
실행하면 인터넷 없이 설치됩니다.

## 4. 실행 중 확인할 것

- 카메라 앞에 아무도 없으면 "카메라 앞에 서 주세요", 인식 직후엔 "인식 중입니다"
  안내가 화면에 크게 표시됩니다 — 이 안내가 사라지면 조작 준비 완료입니다
- 화면 좌상단 통계: 추론 FPS(30 안팎), 카메라 FPS(30 안팎). 시연 화면을 **한 창만**
  열어 두세요 — 창마다 영상 스트림 인코딩이 따로 돌아 여러 창이면 FPS가 떨어집니다
- 화면 상단 "N / 4 단계" 표시로 진행 위치를 알 수 있습니다
- 커서 주변 초록 링(응시 진행) · 화면 맨 위 주황 바(눈감기 진행) — 클릭·뒤로가기가
  얼마나 남았는지 시각으로 보여줍니다
- 화면 하단 "계기판": 입벌림·눈감김 현재값과 평상시 기준값을 함께 보여줍니다 —
  제스처가 안 먹히면 이 수치를 보고 `configs/config.yaml`의 `head_tracker.*`
  여유값(margin)을 조정하세요
- 키오스크 흐름: 증명서 선택 → 주민등록번호 입력(13자리, 뒷자리 자동 마스킹) →
  발급 장수 선택(1~5장) → 발급 확인 → 발급 완료 (뒤로가기는 입력값을 보존한 채
  한 단계씩 되돌아갑니다)

## 5. 자주 막히는 것

| 증상 | 조치 |
|---|---|
| `install.bat`에서 Python을 못 찾음 | Python 3.11.5를 "Add python.exe to PATH" 체크하고 재설치 |
| 카메라가 안 열림/오래 걸림 | `configs/config.yaml`의 `camera.device_id`를 0→1로, `windows_backend`를 msmf↔dshow로 |
| 커서가 예민하게 튐 | `head_tracker.pointer.sensitivity_x`/`sensitivity_y`를 낮춘다 |
| 눈을 안 감았는데 자꾸 뒤로 감 | `head_tracker.eye_close_cancel.close_margin`을 올린다 (2026-07-20부터 고정값이 아니라 "잠금 직후 평상시 눈 상태" 기준 여유값) |
| 입을 안 벌렸는데 선택됨 | `head_tracker.mouth_click.open_margin`을 올린다 (마찬가지로 평상시 기준 여유값) |
| 감아도/벌려도 잘 안 잡힘 | 위 두 항목의 margin을 낮춘다. 계기판의 "기준" 수치와 비교해서 조정하면 빠르다 |

더 자세한 문제 해결은 **[설치가이드.md](설치가이드.md)**의 "G. 문제 해결" 절을 참고하세요.
