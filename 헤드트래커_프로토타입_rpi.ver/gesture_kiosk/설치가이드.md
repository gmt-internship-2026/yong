# 설치가이드 — gesture_kiosk 추론 (라즈베리파이5, 헤드트래커)

다른 라즈베리파이5에 그대로 옮겨 실행하기 위한 가이드입니다.
명령은 전부 **이 폴더(gesture_kiosk) 안에서** 실행합니다.

| 항목 | 기준 |
|---|---|
| 보드 | Raspberry Pi 5 (Cortex-A76 4코어, aarch64) |
| OS | Raspberry Pi OS (Bookworm 이상), 64비트 |
| GPU/가속기 | 불필요 — MediaPipe CPU 추론(XNNPACK, ARM NEON 자동 활용) |
| Python | 3.11.5 — pyenv로 맞춤(아래 A-1). 라즈베리파이OS 릴리스마다 기본 `python3`가 달라 apt만으론 안 됨 |
| 카메라 | USB 웹캠(V4L2, 기본) 또는 공식 카메라 모듈(picamera2, CSI — 성능 최우선이면 권장) |
| 마이크 | 불필요 (입력은 헤드트래커뿐) |

> ⚠ **이 가이드는 실제 라즈베리파이5에서 검증되지 않았다.** 개발 세션이 윈도우 PC에서
> 진행돼 실기 실행이 불가능했다 — MediaPipe·picamera2 공식 문서와 win.ver(윈도우판)의
> 검증된 구조를 기준으로 작성했다. `pip install`이나 카메라 연결에서 막히면
> "G. 문제 해결"과 각 파일 상단 주석의 실기 미검증 표시를 먼저 확인할 것.

## A. 설치 — 3단계

```bash
# 1) 이 폴더를 라즈베리파이로 통째로 복사(scp/USB/git 등)한 뒤, 폴더 안에서:
bash install.sh

# 2) 실행 (다른 기기 브라우저에서 http://<라즈베리파이IP>:5000 을 열면 데모 화면)
bash run.sh
```

`install.sh`가 하는 일: apt 시스템 패키지(espeak-ng·libatlas-base-dev·python3-picamera2 등)
설치 → `--system-site-packages` 가상환경(venv_rpi) 생성 → 고정 버전 패키지 설치 →
얼굴 랜드마크 모델 다운로드(face_landmarker.task, 약 4MB) → 설치 검증(smoke_test).

> 📄 **라이선스**: 판정 엔진은 MediaPipe(Apache-2.0) 단일 모델입니다 — 저작자 표시 등
> 고지 의무가 없습니다(2026-07-22 구글 공식 모델 카드로 재확인). win.ver와 동일 모델
> 파일을 그대로 씁니다.

## A-1. Python 3.11.5 맞추기 (install.sh보다 먼저)

2026-07-22 실기: 라즈베리파이OS 이미지에 따라 기본 `python3`가 3.11이 아니라 3.13
(Debian 13/trixie 기반) 등으로 다를 수 있다 — apt는 그 릴리스가 제공하는 버전 딱
하나만 주기 때문에, win.ver와 정확히 같은 3.11.5를 쓰려면 **pyenv로 소스 빌드**해야
한다. 시스템 기본 `python3`는 건드리지 않고 이 프로젝트 폴더에만 적용된다.

```bash
# 1) 빌드 의존성 (컴파일에 필요, pyenv가 소스에서 직접 빌드한다)
sudo apt update && sudo apt install -y make build-essential libssl-dev zlib1g-dev \
  libbz2-dev libreadline-dev libsqlite3-dev wget curl libncursesw5-dev xz-utils \
  tk-dev libxml2-dev libxmlsec1-dev libffi-dev liblzma-dev

# 2) pyenv 설치
curl https://pyenv.run | bash
echo 'export PYENV_ROOT="$HOME/.pyenv"' >> ~/.bashrc
echo '[[ -d $PYENV_ROOT/bin ]] && export PATH="$PYENV_ROOT/bin:$PATH"' >> ~/.bashrc
echo 'eval "$(pyenv init - bash)"' >> ~/.bashrc
source ~/.bashrc

# 3) 3.11.5 빌드 (라즈베리파이5에서도 컴파일이라 10~20분 정도 걸림)
pyenv install 3.11.5

# 4) 이 폴더에만 지정 (.python-version 파일 생성 — 시스템 python3는 그대로)
cd 헤드트래커_프로토타입_rpi.ver/gesture_kiosk   # 이미 이 폴더면 생략
pyenv local 3.11.5
python3 --version   # 3.11.5 나오면 완료
```

이미 다른 버전으로 `venv_rpi`를 만들어 버렸다면 지우고 `install.sh`를 다시 돌려야
3.11.5가 반영됩니다: `rm -rf venv_rpi && bash install.sh`.

## B. 내부망(오프라인) 반입

win.ver의 `make_offline_bundle.bat` 같은 오프라인 번들 스크립트는 **이 판에는 아직
없습니다**(요청 범위 밖이라 만들지 않음). 인터넷 없는 라즈베리파이에 넣어야 한다면:
1. 인터넷 되는 **같은 아키텍처(aarch64)** 환경에서 `pip download -r requirements.txt -d wheelhouse/`로 휠을 모아 반입 후 `pip install --no-index --find-links wheelhouse -r requirements.txt`
2. `models/weights/face_landmarker.task`를 직접 복사(위 URL은 `scripts/download_weights.py` 참고)
3. apt 패키지(`install.sh` 상단 목록)는 `apt-get download` 또는 로컬 미러로 별도 반입

## D. 회사 키오스크 프로그램(UI) 연동 계약

win.ver와 완전히 동일합니다 — 판정 로직·이벤트 스키마는 플랫폼 무관.

| 방향 | 통로 | 내용 |
|---|---|---|
| 엔진→UI(이산 이벤트) | `event_output` (udp: JSON) 또는 `/data` 폴링 `events` | `select`(입 벌리기/응시) · `go_back`(양 눈 감기 — 뒤로·처음으로 겸용) · `recenter`(입 오므리기 — 커서 재정렬) |
| 엔진→UI(연속 커서) | `/data` 폴링 `status.cursor_x_ratio`/`cursor_y_ratio` | 데모 UI 전용 — 0~1 화면 비율, 추적 끊기면 `null`. **UDP 계약에는 실리지 않음** |
| UI→엔진 | `POST /announce {"text": "..."}` | 커서가 올라간 항목 등 화면 안내를 음성으로 읽어달라는 요청 |

- 회사 UI가 실시간 커서(호버 인터랙션)를 쓸지는 미정 — **UDP로 연속 커서를 전달할지는
  별도 협의 필요**(docs/TODO.md 참고).
- 새 수신 규격(소켓/시리얼 등)이 정해지면 `src/pipeline/event_sender.py`에 Sender 1개를
  추가하고 config `event_output.mode`로 선택합니다 — 파이프라인 수정 불필요.

## E. 동작 기준 (win.ver와 동일 — 판정은 평상시 기준선 상대값)

| 동작 | 판정 | 이벤트 |
|---|---|---|
| 포인터 이동 | 코끝 위치 — 잠금 직후 캘리브레이션한 중심 기준, 안구간거리로 정규화 | (이벤트 없음 — `/data`·비디오 오버레이의 연속 상태) |
| 선택 (빠름) | 입 벌리기 (jawOpen이 "평상시 기준선 + 여유값"을 넘으면) | `select` |
| 선택 (대체) | 커서가 반경 안에 **1.5초** 머무름(응시) | `select` |
| 뒤로(단계별) · 처음으로 | **양 눈 감고 0.6초 버티기** | `go_back` |
| 커서 재정렬 | **입 오므리기** (mouthPucker가 기준선+여유값을 넘으면) | `recenter` |
| 사용자 고정 | 초점 맞은(선명·큰) 얼굴 기준 잠금 | (이벤트 없음 — 상태) |

임계값(margin)은 표정 신호의 특성이라 CPU 플랫폼과 무관해 win.ver 값을 그대로
가져왔습니다 — 설계 이력·제스처 변천 과정은 `src/postprocess/head_tracker.py`
모듈 docstring에 전부 남아 있습니다. 튜닝이 필요하면 계기판(`/data`의 `debug`,
`all_blendshapes` 포함)에서 실제 점수를 보고 `configs/config.yaml`의
`head_tracker.*.open_margin`/`close_margin`을 조정하세요.

## E-1. 카메라 설치 기준

| 항목 | 기준 | 이유 |
|---|---|---|
| 사용자 거리 | 0.5 ~ 0.9 m | 너무 멀면 안구간거리(정규화 자)가 작아져 커서가 둔감해짐 |
| 프레임 구도 | 얼굴 전체 포함 | 코·눈·입 랜드마크가 모두 보여야 함 |
| 카메라 높이 | 사용자 눈높이 근처 | 위/아래 각이 크면 코끝 캘리브레이션 중심이 치우침 |
| 지향 | 사용자 정면 | 측면 각이 크면 안구간거리가 좁게 잡혀 커서가 과민해짐 |
| 조명 | 밝게, 역광 회피 | 역광이면 얼굴 랜드마크·블렌드셰이프 신뢰도가 떨어짐 |

**picamera2(공식 카메라 모듈) 사용 시**: 리본 케이블 방향(파란 면이 보드 쪽) 확인,
`libcamera-hello`(또는 `rpicam-hello`)로 카메라 자체가 먼저 정상 동작하는지 확인한
뒤 `configs/config.yaml`의 `camera.backend: picamera2`로 전환하세요. `raspi-config`에서
카메라 인터페이스가 활성화돼 있어야 합니다.

## F. 개인정보 처리

엔진은 개인정보를 다루지 않습니다 — 입력은 얼굴 랜드마크(좌표 숫자)·블렌드셰이프
(표정 점수)뿐이고, 카메라 프레임은 디스크에 저장하지 않으며, 이벤트에는 동작 이름
(select/go_back/recenter)만 실립니다. 본인확인 절차는 회사 프로그램 담당.
`demo_ui/index.html`의 주민등록번호 입력 키패드도 입력값이 브라우저 안에만 머물고
엔진으로는 전달되지 않습니다(win.ver와 동일 원칙).

## G. 성능 튜닝 — "추론 성능 최대한" 실기 절차

숫자를 바꾸기 전에 반드시 **먼저 측정**하세요:

```bash
venv_rpi/bin/python scripts/benchmark.py
```

1. **카메라 백엔드부터 확인** — `camera.backend: v4l2`로 USB 웹캠을 쓰고 있다면
   `picamera2`(공식 카메라 모듈)로 바꾸는 게 가장 큰 레버입니다. 소프트웨어 리사이즈·
   색변환이 통째로 사라집니다(`src/capture/camera_stream.py` 참고).
2. **`camera.proc_width_px`/`proc_height_px`** — 480x270(현재 기본, 추정값)에서도 30 FPS가
   안 나오면 320x180까지 낮춰 보세요(16:9 유지). 반대로 여유가 있으면 640x360(win.ver
   기준)까지 올려 정확도를 더 확보할 수 있습니다.
3. **`model.infer_scale_ratio`** — proc 해상도를 더 낮추기 싫다면 이 값을 1.0→0.75로
   내려 추론 입력만 추가로 줄입니다(키오스크 거리에선 얼굴이 커서 여유 있음).
4. **`model.num_faces`** — 기본 2(win.ver는 3). 1로는 낮추지 마세요 — person_lock의
   "MediaPipe 자체 랭킹이 잠금을 가로채지 않게" 하는 전제가 깨집니다.
5. **systemd로 실행** — python을 터미널에서 직접 실행하면 라즈베리파이OS가 일반
   사용자의 우선순위 상향을 막아 `realtime_loop.py`의 우선순위 조정이 조용히
   실패합니다. `deploy/gesture_kiosk.service`로 등록하면 `Nice=-5`가 확실히 적용됩니다.
6. **시연 화면은 브라우저 한 창만** — 창마다 MJPEG 인코딩이 따로 돌아 부하가
   배가됩니다(win.ver 실측 — 플랫폼 무관하게 적용됨).
7. **냉각·전력** — RPi5는 서멀 스로틀링이 실제로 성능에 영향을 줍니다. 공식 액티브
   쿨러(팬) 장착을 권장하고, `vcgencmd get_throttled`로 스로틀링 발생 여부를 확인하세요.
   전원은 5V/5A(27W) 공식 어댑터 미만이면 부팅 중 저전압 경고로 성능이 제한됩니다.

## H. 문제 해결

| 증상 | 조치 |
|---|---|
| `install.sh`에서 `pip install` 실패 | mediapipe/opencv-python의 aarch64 휠이 해당 버전에 없을 수 있음 — `requirements.txt` 상단 주석대로 버전 범위를 완화 |
| picamera2 backend에서 `ModuleNotFoundError: picamera2` | venv를 `--system-site-packages` 없이 만들었거나 apt `python3-picamera2` 미설치 — install.sh 재실행 또는 venv_rpi 재생성 |
| 카메라가 안 열림 (`camera.backend: v4l2`) | `v4l2-ctl --list-devices`로 장치 확인, `camera.device_id` 조정 |
| 카메라가 안 열림 (`camera.backend: picamera2`) | `libcamera-hello`(또는 `rpicam-hello`)로 카메라 자체 동작 먼저 확인, `raspi-config`에서 카메라 인터페이스 활성화 확인 |
| 한글 음성이 안 나옴 | `sudo apt install espeak-ng`, 설치 후 `espeak-ng --version`으로 확인. pyttsx3가 자동으로 못 찾으면 `announcer.py`의 `pyttsx3.init()`에 `driverName='espeak'`을 명시해야 할 수 있음(실기 미검증) |
| 추론 FPS가 목표(30)에 크게 못 미침 | 위 "G. 성능 튜닝" 순서대로 진행 — 특히 카메라 백엔드와 proc 해상도 |
| 커서가 상단/한쪽에 눌러붙어 잘 안 움직임 | `head_tracker.pointer.sensitivity_x`/`sensitivity_y`를 낮춘다 (기본 0.8/0.5) |
| 눈을 안 감았는데 자꾸 뒤로 가짐 | `head_tracker.eye_close_cancel.close_margin`을 올린다 (기본 0.3) |
| 입을 안 벌렸는데 자꾸 선택됨 | `head_tracker.mouth_click.open_margin`을 올린다 (기본 0.35) |
| 입을 안 오므렸는데 커서가 자꾸 재정렬됨 | `head_tracker.recenter_gesture.open_margin`을 올린다 (기본 0.43) |
| 다른 사람 얼굴에 반응 | `person_lock.sharpness_weight`를 올려 초점 선명도 비중을 높인다 |
| 커서가 화면 항목과 어긋남 | `camera.proc_width_px/proc_height_px`가 16:9인지 확인 |
| `sudo` 요구로 install.sh가 멈춤 | apt 설치 줄에서 비밀번호를 입력해야 합니다 — 무인 설치가 필요하면 사전에 `sudo -v`로 인증해 두세요 |

## I. 폴더 구조

```
gesture_kiosk/
├─ install.sh / run.sh                # 이식·실행 진입점
├─ deploy/gesture_kiosk.service       # systemd 자동 시작 + 우선순위 보장 (선택)
├─ 설치가이드.md                       # 이 문서
├─ requirements.txt                   # 고정 버전 (재현성, aarch64 휠 확인 필요 — 주석 참고)
├─ configs/config.yaml                # 모든 튜닝값 — 수정은 여기서만
├─ models/weights/                    # face_landmarker.task (빌드 타임 1회 다운로드, git 미포함)
├─ src/
│   ├─ capture/                       # 카메라 캡처 스레드 (v4l2 · picamera2 2백엔드)
│   ├─ inference/                     # 얼굴 랜드마크(FaceLandmarker) 추정
│   ├─ postprocess/                   # 사용자 잠금 + 커서·클릭·뒤로가기·재정렬 판정
│   ├─ announce/                      # 음성 안내 (토크백 TTS, 리눅스는 espeak-ng)
│   ├─ pipeline/                      # 실시간 루프 · 이벤트 전송 · 예시 UI 서버
│   └─ utils/
├─ scripts/                           # run_demo · download_weights · benchmark · smoke_test
├─ demo_ui/                           # 예시 민원발급기 화면 (회사 UI 수령 시 교체)
└─ tests/                             # 단위 테스트 (카메라·모델 없이 실행 가능)
```
