# gesture_kiosk — 헤드트래커 배리어프리 민원발급기 (추론, 라즈베리파이5판)

(주)광명테크 인턴 프로젝트. 카메라 1대로 얼굴 랜드마크를 실시간 인식해 키오스크
프로그램으로 이벤트를 전달한다. **기획서(기획서.docx)의 2.3 디렉터리 구조와 4장
코딩 컨벤션을 따른다.** 이 폴더는 **라즈베리파이5판(rpi.ver)** — 원본은 윈도우판
(`헤드트래커_프로토타입_win.ver/gesture_kiosk`)이며, 2026-07-22 이식.

- 실행 환경: **라즈베리파이5(Cortex-A76, aarch64) + 라즈베리파이OS(Bookworm) + Python 3.11** — GPU 불필요
- 동작 체계(윈도우판과 동일 — 판정 로직은 플랫폼 무관): **코끝=포인터 이동 · 입 벌리기/1.5초
  응시=선택 · 양 눈 감고 0.6초=뒤로(단계별)·처음으로 · 입 오므리기=커서 재정렬**
- 모델: **MediaPipe FaceLandmarker(Apache-2.0) 단일** — 커서·선택·뒤로가기·재정렬이 전부
  얼굴 랜드마크+블렌드셰이프 하나로 판정된다 (학습 0회 스택, CPU 전용)
- 카메라 백엔드 2종(`configs/config.yaml`의 `camera.backend`): **v4l2**(USB 웹캠, 기본) ·
  **picamera2**(공식 카메라 모듈 — libcamera 하드웨어 ISP가 리사이즈·색변환을 대신해
  CPU를 아낀다. 추론 성능을 최대로 뽑고 싶으면 이쪽을 권장)
- 학습(파인튜닝)은 이 폴더 범위 밖 — 추론 전용

## ⚠ 이식 상태 — 반드시 읽을 것

이 판은 **윈도우 개발 PC에서 작성**됐고, 실제 라즈베리파이5 하드웨어에서 실행·측정하지
**못했다**. 코드는 MediaPipe/picamera2 공식 문서 기준으로 정확하게 작성했지만:

- `configs/config.yaml`의 `proc_width_px`(480x270)·`num_faces`(2) 등은 "약한 CPU니까
  낮춰서 시작하자"는 **추정 초기값**이지 실측치가 아니다.
- `requirements.txt`의 mediapipe·opencv-python 버전이 aarch64 리눅스 휠로 존재하는지
  이 세션에서 확인하지 못했다 — `install.sh` 실행 중 실패하면 버전 범위를 완화해서
  재시도할 것(파일 상단 주석 참고).
- `camera.backend: picamera2` 경로는 실기 카메라 모듈로 테스트되지 않았다.

**첫 실행은 `scripts/benchmark.py`로 실측 FPS부터 확인하고, `설치가이드.md`의
"성능 튜닝" 절을 따라 숫자를 실기 기준으로 맞출 것.**

## 빠른 시작 (라즈베리파이5)

```bash
bash install.sh        # apt 의존성 + venv + pip + 모델 다운로드
bash run.sh            # 실행 — 브라우저 http://<라즈베리파이IP>:5000
```

## 인식 동작

| action | 동작 | 판정 방식 | 키오스크 명령 |
|---|---|---|---|
| (연속 상태, 이벤트 아님) | 코끝을 화면 커서로 | 안구간거리 정규화 + EMA 평활 | 포인터 이동(호버) |
| select | **입 벌리기** (즉시) | jawOpen이 평상시 기준선+여유값을 넘으면 | 선택·확인 |
| select | **1.5초 응시**(대체) | 커서가 반경 안에 머무름 — 마스크 등 대비 상시 병행 활성 | 선택·확인 |
| go_back | **양 눈 감고 0.6초 버티기** | eyeBlinkLeft/Right 둘 다 평상시 기준선+여유값 이상 유지 | 뒤로(단계별) · 처음으로(완료 화면) |
| recenter | **입 오므리기** | mouthPucker가 평상시 기준선+여유값을 넘으면 | 커서를 지금 고개 위치로 재정렬 |

윈도우판과 완전히 동일한 임계값을 쓴다 — 표정 판정은 MediaPipe 모델 출력 특성이라
CPU 플랫폼과 무관하기 때문이다. 자세한 판정 로직·설계 이력은
[src/postprocess/head_tracker.py](src/postprocess/head_tracker.py) 모듈 docstring 참고.

## 추론 성능을 위해 이식에서 바꾼 것

1. **카메라 백엔드 선택** — picamera2(공식 카메라 모듈)는 libcamera 하드웨어 ISP가
   목표 해상도·BGR 변환을 직접 뽑아줘 소프트웨어 리사이즈 단계가 통째로 스킵된다
   (`src/capture/camera_stream.py`). CPU를 추론에 더 쓸 수 있게 되는, 이식에서 가장
   큰 성능 레버.
2. **처리 해상도·num_faces 하향** — `configs/config.yaml` 참고(추정 초기값, 실기 튜닝 필요).
3. **systemd 배포용 우선순위 보장** — 리눅스는 일반 사용자가 프로세스 우선순위를
   못 올린다(윈도우와 다름). `deploy/gesture_kiosk.service`의 `Nice=-5`로 확실히 보장.

## 폴더 구조

윈도우판과 동일 — `src/`(추론 파이프라인), `configs/`(설정), `tests/`(단위 테스트,
카메라·mediapipe 없이 동작), `demo_ui/`(예시 UI, 회사 프로그램 인수 전까지),
`scripts/`(실행·벤치마크·모델 다운로드), `deploy/`(systemd 유닛, 라즈베리파이 전용 신설).
