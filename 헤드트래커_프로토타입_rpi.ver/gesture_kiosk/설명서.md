# 설명서 — gesture_kiosk 실행·사용법 (라즈베리파이5)

이 문서 하나로 "어떻게 켜고 어떻게 쓰는지"를 확인할 수 있게 정리했습니다.
설치 절차의 세세한 내용(문제 해결 전체 표, 성능 튜닝)은
**[설치가이드.md](설치가이드.md)**를 참고하세요.

## 1. 이게 뭔가

카메라 1대로 얼굴을 인식해 손 없이 화면을 조작하는 배리어프리 민원발급기 데모입니다.
GPU 없이 CPU만으로 동작합니다. 원본은 윈도우판이며 이 폴더는 **라즈베리파이5판**입니다.

| 동작 | 방법 |
|---|---|
| 포인터 이동 | 코끝을 화면 커서로 — 고개를 움직이면 커서가 따라간다 |
| 선택 | 입 벌리기(즉시) 또는 커서를 1.5초 가만히 두기(응시) — 링 모양 진행 표시가 뜬다 |
| 뒤로(단계별) | 양 눈을 감고 0.6초 버티기 — 화면 상단 진행 바가 채워진다. 직전 단계로 돌아가며
                입력값(주민등록번호·장수)은 그대로 유지된다 |
| 처음으로 | 발급 완료 화면에서는 뒤로가기가 곧 처음으로 (더 되돌아갈 단계가 없어서) |
| 커서 재정렬 | 입을 오므리기(쭉 내밀기) — 자리를 옮기지 않고도 커서 중심을 지금 고개 위치로 다시 잡는다 |

> 입 벌림·눈 감김·입 오므림 판정은 고정 수치가 아니라 **잠금 직후 사용자의 평상시
> 표정을 기준선으로 캡처**해 그로부터 얼마나 벗어났는지로 판정합니다 — 사람마다
> 평상시 표정 편차가 있어도(계기판에 "기준" 수치로 표시) 오탐·미탐이 줄어듭니다.

## 2. 처음 한 번만 — 설치

이 폴더(`gesture_kiosk`) 안에서:

```bash
bash install.sh
```

python3 확인 → apt 시스템 패키지(espeak-ng 등) 설치 → 가상환경(`venv_rpi`) 생성 →
패키지 설치 → 얼굴 랜드마크 모델 다운로드(`models/weights/face_landmarker.task`) →
설치 검증(스모크 테스트)까지 자동으로 됩니다.

## 3. 실행하는 법 (전부)

### 3-1. 데모 실행 (가장 많이 쓰는 것)

```bash
bash run.sh
```

같은 네트워크의 다른 기기 브라우저에서 `http://<라즈베리파이IP>:5000` 접속 →
실시간 캠 화면 위에 키오스크 메뉴가 얹혀 나옵니다. 종료는 이 창에서 `Ctrl+C`.
라즈베리파이 자체 화면에서 보려면 라즈베리파이의 브라우저로 `http://localhost:5000`.

### 3-2. 헤드리스 실행 (UI 없이 이벤트만)

회사 프로그램과 실제로 연동할 때처럼, 화면 없이 판정 결과만 `event_output` 설정대로
내보냅니다(기본은 콘솔 로그). 실제 무인 키오스크 배포는 이 모드를
`deploy/gesture_kiosk.service`(systemd)로 부팅 시 자동 실행하는 걸 권장합니다.

```bash
bash run.sh --headless
```

### 3-3. 직접 파이썬으로 실행 (run.sh 안 거치고)

`venv_rpi`를 이미 만든 상태라면:

```bash
source venv_rpi/bin/activate
python scripts/run_demo.py
python scripts/run_demo.py --headless
```

활성화 없이 한 줄로 실행하려면 가상환경의 python을 직접 지정합니다:

```bash
venv_rpi/bin/python scripts/run_demo.py
```

### 3-4. 단위 테스트 (카메라·모델 없이 판정 로직만 검증)

```bash
venv_rpi/bin/python -m unittest discover tests -v
```

### 3-5. 설치 검증 (스모크 테스트)

카메라 없이, 패키지가 제대로 깔렸는지 + 더미 프레임으로 얼굴 랜드마크 모델이
로드되는지 확인합니다. `install.sh` 마지막 단계에서 자동으로도 실행됩니다.

```bash
venv_rpi/bin/python scripts/smoke_test.py
```

### 3-6. 추론 FPS 벤치마크 (실기 성능 측정, 카메라 필요) — **가장 먼저 해볼 것**

```bash
venv_rpi/bin/python scripts/benchmark.py                    # 더미 프레임 1000장
venv_rpi/bin/python scripts/benchmark.py --source camera    # 실제 카메라로 측정
venv_rpi/bin/python scripts/benchmark.py --frame-count 300  # 프레임 수 조절
```

`configs/config.yaml`의 처리 해상도·`num_faces`는 라즈베리파이5 실기 미검증
추정값입니다 — 이 벤치마크 결과를 보고 [설치가이드.md](설치가이드.md)의
"G. 성능 튜닝" 절대로 숫자를 맞추세요.

### 3-7. 얼굴 랜드마크 모델만 다시 받기

```bash
venv_rpi/bin/python scripts/download_weights.py
```

## 4. 실행 중 확인할 것

- 카메라 앞에 아무도 없으면 "카메라 앞에 서 주세요", 인식 직후엔 "인식 중입니다"
  안내가 화면에 크게 표시됩니다 — 이 안내가 사라지면 조작 준비 완료입니다
- 화면 좌상단 통계: 추론 FPS, 카메라 FPS. 시연 화면을 **한 창만** 열어 두세요 —
  창마다 영상 스트림 인코딩이 따로 돌아 여러 창이면 FPS가 떨어집니다
- 화면 상단 "N / 4 단계" 표시로 진행 위치를 알 수 있습니다
- 커서 주변 초록 링(응시 진행) · 화면 맨 위 주황 바(눈감기 진행)
- 화면 하단 "계기판": 입벌림·눈감김·입오므림 현재값과 평상시 기준값을 함께
  보여줍니다 — 제스처가 안 먹히면 이 수치를 보고 `configs/config.yaml`의
  `head_tracker.*` 여유값(margin)을 조정하세요
- 키오스크 흐름: 증명서 선택 → 주민등록번호 입력(13자리, 뒷자리 자동 마스킹) →
  발급 장수 선택(1~5장) → 발급 확인 → 발급 완료

## 5. 자주 막히는 것

| 증상 | 조치 |
|---|---|
| `install.sh`에서 pip 실패 | mediapipe/opencv-python aarch64 휠 확인 — `requirements.txt` 상단 주석 |
| 카메라가 안 열림 | v4l2면 `v4l2-ctl --list-devices`로 확인, picamera2면 `libcamera-hello`로 카메라 자체부터 확인 |
| 커서가 예민하게 튐 | `head_tracker.pointer.sensitivity_x`/`sensitivity_y`를 낮춘다 |
| 눈을 안 감았는데 자꾸 뒤로 감 | `head_tracker.eye_close_cancel.close_margin`을 올린다 |
| 입을 안 벌렸는데 선택됨 | `head_tracker.mouth_click.open_margin`을 올린다 |
| 입을 안 오므렸는데 커서가 재정렬됨 | `head_tracker.recenter_gesture.open_margin`을 올린다 |
| 추론 FPS가 낮음 | [설치가이드.md](설치가이드.md) "G. 성능 튜닝" 절 순서대로 (카메라 백엔드 전환이 최우선) |

더 자세한 문제 해결은 **[설치가이드.md](설치가이드.md)**의 "H. 문제 해결" 절을 참고하세요.

## 6. 파일 어디 있나

| 무엇을 바꾸고 싶을 때 | 파일 |
|---|---|
| 제스처 민감도·임계값·처리 해상도 튜닝 | `configs/config.yaml` |
| 커서·클릭·뒤로가기·재정렬 판정 로직 | `src/postprocess/head_tracker.py` |
| 카메라 백엔드(v4l2/picamera2) | `src/capture/camera_stream.py` |
| 예시 키오스크 화면(메뉴·키패드) | `demo_ui/index.html` |
| 회사 프로그램과의 연동 규격 | `src/pipeline/event_sender.py`, [설치가이드.md](설치가이드.md)의 "D. 회사 키오스크 프로그램(UI) 연동 계약" 절 |
| 부팅 자동 시작·우선순위 보장 | `deploy/gesture_kiosk.service` |
